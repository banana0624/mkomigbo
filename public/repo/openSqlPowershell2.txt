## 1. Quick one-off connection (recommended)

From PowerShell:

```powershell
# 1) Go to your MariaDB bin folder
cd "F:\xampp\MariaDB 11.8\New Folder\bin"

# 2) Connect as root (no password) on port 3306:
.\mysql.exe -u root --port=3306 --protocol=TCP
```

If **root** has a password:

```powershell
cd "F:\xampp\MariaDB 11.8\New Folder\bin"

.\mysql.exe -u root -p --port=3306 --protocol=TCP
```

You’ll see:

```text
Enter password:
```

Once inside the MariaDB prompt:

```sql
USE mkomigbo;
SHOW TABLES;
SOURCE F:/xampp/htdocs/mkomigbo/project-root/private/db/seed_roles_users.sql;
```

and exit when done:

```sql
EXIT;
```

## 4. Optional PowerShell shortcut command

Update the helper function to use the **new MariaDB path**.

In PowerShell:

```powershell
notepad $PROFILE
```

Add or update this function:

```powershell
function mkomigbo-db {
    param(
        [string]$User = "root",
        [int]$Port = 3306
    )

    $pw = Read-Host "MariaDB password for $User (leave blank if none)" -AsSecureString
    $bstr  = [System.Runtime.InteropServices.Marshal]::SecureStringToBSTR($pw)
    $plain = [System.Runtime.InteropServices.Marshal]::PtrToStringAuto($bstr)

    & "F:\xampp\MariaDB 11.8\New Folder\bin\mysql.exe" `
        --host=127.0.0.1 `
        --port=$Port `
        --user=$User `
        --password=$plain `
        mkomigbo
}
```

Save and close, then open a **new** PowerShell window and use:

```powershell
mkomigbo-db               # default: root on 3306
mkomigbo-db -User uzoma   # connect as uzoma
```

Inside MariaDB you can then run anything you need:

```sql
SHOW TABLES;
SOURCE F:/xampp/htdocs/mkomigbo/project-root/private/db/seed_roles_users.sql;
```
